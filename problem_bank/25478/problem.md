# 25478. 수신기 배정

## Constraints

- **Difficulty**: D7 (VH)
- **Time Limit**: 140 test cases combined — C: 8s / C++: 8s / Java: 12s / Python: 16s
- **Memory Limit**: Heap + static memory 256MB, Stack 1MB
- **Point**: 200
- **Accuracy**: 7.41%

## Problem Description

어느 회사에는 1부터 X까지의 번호가 붙은 X개의 사물함이 있다. 당신은 총 N(1 ≤ N ≤ X)개의 수신기를 사물함들에 넣고자 한다.

각각의 수신기는 사물함 하나에 들어 있어야 하고, 하나의 사물함에는 최대 한 개의 수신기만 들어 있어야 한다.
따라서, 수신기를 배정할 수 있는 모든 방법의 수는 X(X-1)⋯(X-N+1)가지이다. 편의상 수신기에도 1 이상 N 이하의 번호가 붙어 있다고 하자.

사물함은 번호가 증가하는 순으로 1미터 간격으로 놓여 있으므로, i번 사물함과 j번 사물함 사이의 거리는 |i – j|미터이다.
두 수신기 사이에 통신이 가능할 필요충분조건은, 수신기 사이의 거리(정확히는, 수신기가 들어 있는 사물함 사이의 거리)가 D미터 이하인 것과 같다.

M×M 크기의 0 또는 1로 구성된 행렬 A가 주어진다. A는 대칭행렬이며, 대각선에는 모두 1이 적혀 있다.
행렬 A는 수신기끼리 통신이 가능한지의 여부를 나타낸다. 정확히는, 모든 1 ≤ i < j ≤ M에 대해, A의 i행 j열이 '1'이면 i번 수신기와 j번 수신기끼리는 통신 가능해야 하고, '0'이면 통신이 불가능해야 한다.

수신기를 배정할 수 있는 모든 방법 가운데, 수신기끼리의 통신 가능성이 윗 문단에서 제시한 대로 행렬 A와 합치하는 경우의 수를 10^9+7로 나눈 나머지를 출력하라.

## Input

각 테스트 케이스는 아래와 같이 구성된다.

- 첫 번째 줄에는 세 개의 정수 N, X, D (1 ≤ X ≤ 100, 1 ≤ N ≤ min(X, 50), 1 ≤ D ≤ 8)이 공백 하나씩을 사이로 두고 주어진다.
- 다음 N개의 줄에는 행렬 A가 주어진다. 각각의 줄에는 N개의 문자(0 또는 1)가 주어지며, i번째 줄의 j번째 문자는 A(i,j)이다.
  A는 대칭행렬이고, 대각선에는 1만이 적혀 있음이 보장된다.

## Output

각 테스트 케이스에 대해, 수신기를 배정할 수 있는 모든 방법 가운데, 수신기끼리의 통신 가능성이 행렬 A와 합치하는 경우의 수를 10^9+7로 나눈 나머지를 출력하라.
