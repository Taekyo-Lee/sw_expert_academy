# 24424. 올바른 경로

- **Difficulty**: D6 (VH)
- **Point**: 150
- **Accuracy**: 23.53%

## Constraints

- **Time**: 28 test cases combined — C: 20s / C++: 20s / Java: 30s / Python: 30s
- **Memory**: Heap + static memory ≤ 256MB, Stack ≤ 1MB

## Problem

알파벳 소문자 a, b, c, …, z로 구성된 모든 문자열의 집합을 생각하자.

집합 A, B, C, …, Z는 이 집합의 부분집합이며, 무한집합일 수도 있고 유한집합일 수도 있다.

이 집합들은 아래와 같은 두 종류의 규칙으로 정의된다. 아래 표기에서 X, Y, Z는 임의의 두 집합 (서로 같을 수 있음), x는 임의의 알파벳 소문자를 의미한다.

- X -> YZ : s₁ ∈ Y, s₂ ∈ Z이면 s₁s₂ ∈ X이다. 여기서 s₁s₂는 문자열 s₁와 문자열 s₂를 이어붙인 것이다.
- X -> x : x ∈ X이다.

A, …, Z를 처음에는 공집합으로 두고, 위와 같은 형태로 주어진 규칙들을 계속 반복 적용시키면서 새로운 원소들을 추가할 수 있다.

이렇게 규칙들을 무한히 반복 적용했을 때의 집합을 A, …, Z라고 하자.

이해를 돕기 위해 몇 가지 예를 들어 보자.

**예 1:** A -> a, A -> AA
- A = {"a", "aa", "aaa", "aaaa", …}
- B, C, .., Z는 모두 공집합이다.

**예 2:** A -> AA
- A는 공집합이다. (처음에 공집합이므로, A에 원소를 추가할 수가 없다.)

**예 3:** A -> a, A -> AA, B -> b, B -> BB, C -> AB
- A = {"a", "aa", "aaa", "aaaa", …}
- B = {"b", "bb", "bbb", "bbbb", …}
- C = {(a만으로 구성된 문자열) + (b만으로 구성된 문자열)}

**예 4:** S -> SS, S -> LX, X -> SR, L -> a, R -> b, X -> LR
- 첫 조건을 해석해 보면, s₁, s₂ ∈ S이면 s₁s₂ ∈ S이다.
- 두 번째 – 네 번째 조건을 해석해 보면, s ∈ S이면 aSb ∈ S임을 알 수 있다.
- 세 번째 – 다섯 번째 조건을 해석해 보면, ab ∈ S임을 알 수 있다.
- 따라서, S는 a를 '(', b를 ')'로 치환했을 때 모든 올바른 괄호 문자열로 구성된 집합임을 알 수 있다.
- L = {"a"}, R = {"b"}이다.
- X = {(올바른 괄호 문자열) + "b"} ∪ {"ab"}이다.

이러한 상황에서 방향 그래프가 주어진다. 방향 그래프의 각 간선에는 알파벳 소문자 한 글자가 적혀 있다.

방향 그래프의 정점 x에서 정점 y로 가는 모든 경로(단순 경로가 아니어도 됨)를 고려해 보자.

이 경로들 중, 경로에 나온 문자들을 순서대로 나열한 문자열이 집합 S (문제 초반에 정의한 집합)의 원소인 것이 존재하는지 판단하고, 존재한다면 그 중 가장 짧은 문자열의 길이를 구하는 프로그램을 작성하라.

## Input

첫 번째 줄에 테스트 케이스의 수 T가 주어진다. 각 테스트 케이스는 아래와 같이 구성된다.

첫 번째 줄에는 규칙의 수 R(1 ≤ R ≤ 100)이 주어진다.

다음 R개의 줄에는 "(알파벳 대문자) -> (알파벳 대문자)(알파벳 대문자)" 또는 "(알파벳 대문자) -> (알파벳 소문자)" 꼴의 문자열이 한 줄에 하나씩 주어지며, 각각은 집합 A, …, Z를 구성하는 규칙을 의미한다.

그 다음 줄에는 네 개의 정수 n, m, x, y (1 ≤ n ≤ 26, 0 ≤ m ≤ n², 1 ≤ x, y ≤ n)가 주어진다. 이는 그래프의 정점이 n개, 간선이 m개이며, 경로의 출발 정점은 x, 도착 정점은 y임을 의미한다.

다음 m개의 줄에는 두 정수 uᵢ, vᵢ (1 ≤ uᵢ, vᵢ ≤ n)와 알파벳 소문자 cᵢ가 공백 하나씩을 사이로 두고 주어진다. 이는 방향 그래프에 정점 uᵢ에서 정점 vᵢ를 향하는 방향성 간선이 있고, 이 간선에는 알파벳 소문자 cᵢ가 적혀 있다는 뜻이다.

- 그래프의 각 정점에는 1 이상 n 이하의 자연수 번호가 붙어 있다.
- 그래프의 각 간선은 자기 자신을 이을 수도 있다.
- 정점 u에서 정점 v를 향하는 간선과 정점 v에서 정점 u를 향하는 간선이 둘 다 존재할 수도 있다.
- 중복 간선은 존재하지 않는다.
- 정점 x에서 정점 y로 가는 경로가 존재하지 않을 수도 있다.

## Output

각 테스트 케이스에 대해, 경로에 나온 문자들을 순서대로 나열한 문자열이 집합 S의 원소인 것이:

- 존재한다면, 그 중 가장 짧은 문자열의 길이를 10⁹+7로 나눈 나머지를 출력한다.
- 존재하지 않는다면, -1을 출력한다.
