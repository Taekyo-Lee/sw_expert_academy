# Plan 1 — Problem 25699: Tiling (타일링)

## Problem Analysis

We have an infinite 2D grid tiled by translational copies of a single simple rectilinear polygon (polyomino). All copies are identical (no rotation/reflection) and placed by pure translation. The tiling is *lattice-periodic*: there exist two linearly independent translation vectors **v1** and **v2** such that shifting the entire tiling by any integer combination `a*v1 + b*v2` maps it onto itself.

We are given an H x W window (H, W <= 16) into this tiled grid, where each cell is either '#' or '.'. We must find the **minimum area** of a polyomino tile that could produce the observed window.

The area of the tile equals `|det(v1, v2)|`, the area of the fundamental domain of the lattice.

## Key Observations

1. **Lattice structure**: A translational tiling of the plane by a single polyomino is always a lattice tiling. The tile has area equal to the determinant of the lattice basis. So we need to find the smallest-determinant lattice such that the window is consistent with the lattice periodicity.

2. **Consistency condition**: Given a lattice L generated by v1, v2, two cells (r1, c1) and (r2, c2) in the window must have the same color whenever `(r2 - r1, c2 - c1)` is a vector in L. Equivalently, for every lattice vector (dr, dc) in L, if both (r, c) and (r+dr, c+dc) are in the window, they must have the same color.

3. **Hermite Normal Form (HNF)**: Every 2D integer lattice has a unique HNF basis:
   - v1 = (a, 0)
   - v2 = (b, d)
   - where a > 0, d > 0, 0 <= b < a, and det = a * d = A (the area)

   For a given area A, enumerate all HNF bases by iterating over divisor pairs (a, d) with a * d = A, and for each pair, b in [0, a-1].

4. **Precompute compatibility**: For each possible shift vector (dr, dc) with |dr| < H and |dc| < W, precompute whether the shift is "compatible" — i.e., for every cell (r, c) where both (r, c) and (r+dr, c+dc) are in the window, they have the same color. This is an O(H * W) check per vector, and there are O(H * W) vectors, so O((HW)^2) = O(65536) total — negligible.

5. **Fundamental domain existence**: For any 2D integer lattice, a simply-connected rectilinear fundamental domain (polyomino) always exists. So if a lattice is compatible with the window, a valid simple rectilinear polygon tile of that area exists.

## Algorithm Choice

**Exhaustive search over lattice areas using HNF enumeration with precomputed compatibility.**

- Precompute compatibility in O((HW)^2) time.
- For each area A from 1 to H*W (at most 256), enumerate all HNF bases with determinant A.
- For each HNF basis, enumerate all lattice vectors in the window range and check compatibility in O(1) per vector.
- Return the first A for which at least one valid HNF basis exists.

**Complexity justification**: The total number of lattice vectors to check across all HNFs and all areas is bounded by approximately sum_{A=1}^{256} sigma(A) * 4HW/A, which is roughly 1.5 * 10^6. Each check is O(1) with precomputation. This is extremely fast, well within 30 seconds even for 201 test cases.

## Step-by-Step Approach

### Step 0: Precompute compatibility table

For each (dr, dc) with `-(H-1) <= dr <= (H-1)` and `-(W-1) <= dc <= (W-1)`:
- Set `compat[dr][dc] = True` if for ALL cells (r, c) in the grid, whenever (r+dr, c+dc) is also in the grid, `grid[r][c] == grid[r+dr][c+dc]`.
- Note: `compat[0][0]` is always True.

Store as a 2D boolean array (using offset indexing to handle negative indices, or a dictionary).

For each A:

### Step 2: Enumerate HNF bases with determinant A

For each divisor `a` of A (where a > 0):
- Set `d = A // a`
- For each `b` in range(0, a):
  - The HNF basis is v1 = (a, 0), v2 = (b, d)

### Step 3: Check all lattice vectors in window range

For HNF basis v1=(a,0), v2=(b,d):
- Enumerate all integer pairs (m, n) such that the lattice vector `(m*a + n*b, n*d)` falls in the range `[-(H-1), H-1] x [-(W-1), W-1]`.
  - n ranges from `ceil(-(W-1)/d)` to `floor((W-1)/d)`
  - For each n, m ranges appropriately so that `m*a + n*b` is in `[-(H-1), H-1]`
- Skip the zero vector (0, 0).
- For each non-zero lattice vector (dr, dc) in range, check `compat[dr][dc]`.
- If ALL such vectors are compatible, this HNF basis is valid for area A.

### Step 4: Return the answer

If any HNF basis is valid for area A, output A and move to the next test case.

## Edge Cases

1. **All cells same color** (e.g., all '#' or all '.'): Answer is 1, since the trivial lattice (1,0),(0,1) is always compatible.
2. **H=1 or W=1**: The grid is a single row or column. The lattice search still works; many shift vectors have empty overlap and are trivially compatible.
3. **Checkerboard-like pattern**: The answer could be 2, with a lattice like (2,0),(1,1).
4. **Entire window is a single tile repetition**: The answer equals the area of the smallest repeating unit.
5. **Maximum area = H*W = 256**: If no smaller lattice works, the answer could be up to 256 (though the problem guarantees a valid tiling exists, so some A <= 256 must work).

## Complexity Analysis

- **Precomputation**: O(H^2 * W^2) = O(65536) per test case.
- **Main search**: For each area A, the number of HNF bases is sigma_0(A) * A / sigma_0(A) ... more precisely, the total number of (a, b, d) triples is sum_{a|A} a = sigma_1(A). The number of lattice vectors per HNF in range is O(HW/A). Total lattice vector checks across all A: sum_{A=1}^{256} sigma_1(A) * HW/A. This is bounded by about O(HW * 256 * log(256)) ~ O(500,000).
- **Overall per test case**: O(H^2 * W^2 + 500,000) ~ O(500,000) which is very fast.
- **201 test cases**: ~10^8 total operations at most, feasible in 30s with PyPy.
- **Space**: O(H * W) for the grid and compatibility table — negligible.
