# 25333. 시너지 효과

- **Difficulty**: D7 (VH)
- **Point**: 200
- **Accuracy**: 33.33%

## Constraints

- **Time**: 68 test cases combined — C: 20s / C++: 20s / Java: 25s / Python: 30s
- **Memory**: Heap + static memory ≤ 256MB, Stack ≤ 1MB

## Problem

1부터 N까지의 번호가 붙은 N개의 아이템이 있다. i(1 ≤ i ≤ N)번째 아이템의 잠재력은 cᵢ이다.

cᵢ = -1이면 i번 아이템은 빈 상자로, 의미가 없다.

cᵢ ≥ 0이면 i번 아이템은 **유의미한** 아이템이다.

아이템을 사용하려면 각각의 아이템을 정점으로 하는 트리(무향 연결 그래프)를 만들어야 한다.

이렇게 트리가 만들어지면, 모든 유의미한 아이템 가운데, 트리에서 인접한 유의미한 아이템이 존재하는 아이템들만 사용할 수 있다. (쉽게 말해, "고립되지 않은" 유의미한 아이템만 사용할 수 있는 것이다.)

이 때 얻을 수 있는 효과는, 사용할 수 있는 모든 유의미한 아이템들의 잠재력의 합이다.

모든 서로 다른 트리에 대해, 얻을 수 있는 효과가 얼마인지를 계산할 수 있을 것이다.

이 때, 효과가 K 이하인 트리의 개수를 10^9+7로 나눈 나머지를 구하는 프로그램을 작성하라.

정점 집합이 같은 두 트리가 서로 다르다는 것은, 어떤 간선이 한 트리에는 존재하고 다른 트리에는 존재하지 않는다는 것이다.

## Input

첫 번째 줄에 테스트 케이스의 수 T가 주어진다. 각 테스트 케이스는 두 개의 줄로 구성된다.

첫 번째 줄에는 두 정수 N과 K (1 ≤ N ≤ 40, 1 ≤ K ≤ 10^9)가 공백 하나를 사이로 두고 주어진다.

두 번째 줄에는 N개의 정수 c₁, c₂, ···, cₙ (-1 ≤ cᵢ ≤ 2.5×10^7)가 공백 하나씩을 사이로 두고 주어진다.

## Output

각 테스트 케이스에 대해, 효과가 K 이하인 트리의 개수를 10^9+7로 나눈 나머지를 출력한다.

## Note

- 예제 1: 모든 트리의 개수는 5^(5-2) = 125가지이다.
- 예제 2: 맨 왼쪽 트리의 효과는 0, 가운데와 맨 오른쪽 트리의 효과는 c₁ + c₂ = 1+2 = 3이다.
- 예제 3: (트리 그림은 문제 원본 참조)

## Sample

### Input
```
4
5 0
0 0 0 0 0
4 3
1 2 4 -1
4 1
0 1 1 -1
6 3
-1 1 3 5 -1 -1
```

### Output
```
125
3
5
324
```
